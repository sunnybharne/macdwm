Troubleshooting¶

Common issues and solutions for macdwm. Most problems are user error.

Installation Issues¶

"Failed to create event tap"¶

This error means macdwm can't create a global event tap, usually because you didn't give it permissions.

Solution: 1. Open System Settings → Privacy & Security → Accessibility 2. Click the + button and add your terminal application 3. Make sure the toggle is enabled (green) 4. Restart your terminal application 5. Run macdwm again

"Permission denied" when running¶

The binary doesn't have execute permissions.

Solution:

chmod +x macdwm
./macdwm

Runtime Issues¶

Menu bar icon not appearing¶

The terminal icon should appear in your menu bar when macdwm is running.

Troubleshooting steps: 1. Check if macdwm is running: bash ps aux | grep macdwm

  1. Look for error messages: bash # Run in foreground to see output ./macdwm

  2. Check Console.app for error messages:

  3. Open Console.app
  4. Search for "macdwm"
  5. Look for error messages

Hotkeys not working¶

The keyboard shortcuts are not responding.

Troubleshooting steps: 1. Check if macdwm is enabled: - Click the menu bar icon - Make sure "Enabled" is checked

  1. Verify Accessibility permissions:
  2. System Settings → Privacy & Security → Accessibility

  3. Make sure your terminal app is listed and enabled

  4. Test with different applications:

  5. Try tiling different windows

  6. Test with different applications

  7. Check for hotkey conflicts:

  8. System Settings → Keyboard → Shortcuts
  9. Look for conflicting shortcuts

App launcher not working¶

The Cmd+1-5 hotkeys are not launching applications.

Troubleshooting steps: 1. Verify application paths: bash # Check if applications exist ls -la "/Applications/Firefox.app" ls -la "/Applications/iTerm.app"

  1. Test manual launch: bash # Try launching manually open "/Applications/Firefox.app"

  2. Check bundle IDs: bash # Get bundle ID for an app mdls -name kMDItemCFBundleIdentifier "/Applications/Firefox.app"

Window tiling not working¶

Windows are not resizing or repositioning.

Troubleshooting steps: 1. Make sure window is focused: - Click on the window before using tiling hotkeys - Use Cmd+Tab to switch to the window

  1. Test with different applications:
  2. Some apps may not support window manipulation

  3. Try with Finder, Safari, or Terminal

  4. Check window properties:

  5. Some windows may be non-resizable
  6. Try with a resizable window

Getting Help¶

Common solutions¶

  1. Restart macdwm - Solves most temporary issues
  2. Re-grant permissions - Fixes accessibility issues
  3. Check for conflicts - Resolves hotkey problems
  4. Update system - Fixes compatibility issues

Reporting issues¶

When reporting issues, include:

  1. macOS version: sw_vers
  2. Architecture: uname -m
  3. Error messages: From Console.app or terminal
  4. Steps to reproduce: Detailed reproduction steps
  5. Expected behavior: What should happen
  6. Actual behavior: What actually happens

Before you ask for help¶

  1. Read this troubleshooting guide
  2. Check if the issue is already reported on GitHub
  3. Make sure you've given macdwm proper permissions
  4. Try restarting the application
  5. Check the console for error messages

Most issues are caused by not giving macdwm Accessibility permissions. Make sure you've done this step properly.
